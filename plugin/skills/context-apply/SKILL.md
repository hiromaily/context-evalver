# context-apply

Loads the staging file written by `/context-draft`, displays all pending patches with their metadata, requests explicit user confirmation before writing any files, applies each patch to disk, and runs `git diff` to show the result. Requires a prior `/context-draft` run.

## Prerequisites

Run `/context-draft` first. This command reads the staging file (`drafts/{session_id}.json`) produced by that run. If no staging file is found, the command explains how to proceed.

## What it does

1. Loads the draft staging file for the current session
2. Displays each patch with confidence, severity, evidence count, and the unified diff
3. Requests explicit confirmation before writing any files
4. Applies each patch: creates new files or updates existing ones
5. Runs `git diff` and displays the changes
6. If `auto_pr = true` in `.context-evalver.json`, creates a `git commit` with a descriptive message; otherwise, prints a manual `git commit` command
7. Removes the staging file after apply

## Usage

```
node dist/context-apply.js
```

This skill delegates to the compiled `dist/context-apply.js` Node.js script. On failure to write an individual file, it reports the error and continues applying remaining patches.
